## Entry 1 — Session bootstrap (2026-06-05)

- **Mode:** Create. **Working path:** Design handoff (producer-shaped prompt → external visual-design tool; EXPERIENCE.md follows via Update).
- **Scope:** Full platform — player-facing in-game RTS HUD **and** creator-facing Creation Suite, plus shell/menus (title, lobby/matchmaking, content browser).
- **Stakes:** Commercial — premium one-time-purchase PC title (Steam + direct DRM-free), solo dev (Alec). Road to 1.0.
- **Sources confirmed:** `Project_Chimera_GDD.md`, `Snapshot.md`, `STATUS.md` (deprecated tracker), PRD `prd-Project_Chimera-2026-06-05/` (prd.md + addendum). Extracted inline (modest size), not via subagent.
- **Engine/UI system:** Godot 4.6.2 **Control nodes** at runtime (same system as the Godot editor — supports `GraphEdit`/`GraphNode` for visual scripting). Both spines inherit from Godot Control defaults; DESIGN tokens extend, EXPERIENCE specifies behavioral delta.

### Facts inherited from sources (not decisions — context for the spines)

- **Form-factor:** PC desktop only (Windows primary; Linux server). Keyboard + mouse. No console/mobile/VR/web. (GDD §2, PRD §2.4)
- **3D art direction (fixed):** stylized low-poly, clean geometric shapes, soft/pastel colors, slight cel-shading; reference band = Mindustry (utilitarian clarity) ↔ Northgard (painterly warmth). (GDD §1)
- **Player archetypes / personas:** Commander (pure player, never opens editor — editor must be invisible to them), Architect (builder, primary persona), Tinkerer (both). Developer-as-primary-user is the load-bearing 1.0 persona. (GDD §1, PRD §2)
- **Named user journeys (PRD §2.5):** UJ-1 Build a custom game · UJ-2 Co-create with AI · UJ-3 Fast edit→play loop · UJ-4 Publish a scenario · UJ-5 Discover & play community content · UJ-6 Play a multiplayer match.
- **Creation-suite UX heritage (GDD §5):** WC3 World Editor (one entity, one view, all data consolidated — the model to follow); SC2 Galaxy Editor (scattered 5-editor data = the anti-pattern); Mario Maker (mandatory proof-of-play gate); Dreams (hover tooltip on every control); Fortnite Creative (creation invisible to the 99.8% who don't create).
- **Layered complexity / progressive disclosure (pillar):** every creator system needs a simple mode (presets/dropdowns/wizards) AND an advanced mode (raw JSON / visual scripting). Trigger tiers T1 preset → T2 ECA → T3 GraphEdit node graph → T4 natural-language AI, all one DSL.
- **Game feel / juice (framework guarantee, GDD §3 / PRD FR-12a):** a default combat-feedback profile ships (hit particles, impact sound, screen shake, hit-freeze, death effect); creators override per unit/ability.
- **Accessibility floor (PRD FR-51, 1.0 baseline):** remappable keys, colorblind-safe team colors, UI scaling, subtitles for any VO.
- **As-built player HUD (STATUS/Snapshot):** resource counters, supply, minimap (bottom-right, FoW + dots, click-to-pan), command card (per-building production, train progress, `[need: X]` prereq tags), selection (box/click + control groups), build menu, rally markers, stall indicator banner ("Waiting for peer…"), victory/defeat + score-summary cards, settings panel.
- **As-built creation-suite UI (programmatic Control panels):** terrain brush panel, entity placer palette (top-right), start-position/resource-node/win-condition setters, trigger editor panel (L key), AI map generator panel (M key), lobby UI (N key), content browser (Local + mod.io).

## Entry 2 — Visual identity direction (2026-06-05, Alec)

Four directional decisions captured to seed the design-handoff prompt. Canonical here; DESIGN.md
Brand & Style seeded as draft; reconciled with producer output at Finalize.

- **D1 — HUD diegesis: Clean non-diegetic overlay.** Flat UI panels float over the 3D world (SC2 /
  classic-RTS convention). Chrome is honest UI, not in-world. Rationale: clarity at zoomed-out RTS
  camera distances; matches the data-driven engine identity.
- **D2 — UI aesthetic: Stylized low-poly echo.** UI geometry echoes the 3D art — faceted/angular
  panel shapes, flat cel-shaded color blocks, geometric icons. UI and world read as one piece.
  (Compatible with D1: angular *flat overlay* panels, not in-world diegetic surfaces.)
- **D3 — Color: Dark theme primary.** Dark UI is the default (long-session creation-tool
  convention, eye comfort), light text, accent pops. Optional light mode is a later consideration.
  **Faction/team colors are reserved for in-world unit identity** and must not collide with chrome
  accents. Brand accent color: **producer to propose** within the dark theme (no brand color
  established yet). `[ASSUMPTION: single accent; revisit if a second is needed for create-vs-play
  mode signalling.]`
- **D4 — Handoff surface batch (all four):** In-game RTS HUD · Creation Suite shell (incl. the
  consolidated Unit Card Editor) · Shell & menus (title, main menu, settings, lobby/matchmaking,
  victory/defeat) · Content browser (grid, detail, publish, profiles).

### Tension noted for Finalize
- D1 (non-diegetic, clarity-first) + D2 (low-poly echo, themed geometry) must be balanced so the
  faceted styling never costs legibility at RTS distances. Mock review should check the HUD at
  zoomed-out scale specifically.
